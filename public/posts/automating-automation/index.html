Automating Automation

February 11, 2025 · 2 min · 350 words · Me

I finally found my first use case for a large language model chatbot.

I mentioned in my first post that I would use Google’s chatbot Gemini to help set up the hugo.yaml config file for this site. But, that didn’t really work out. Gemini was able to help by defining a few key parameters. It could not, however, create a .yaml file through prompting alone. I ended up having to do it the old fashioned way – by referencing the Hugo docs.

My First Use Case#

The initial problem#

I am enrolled in school, going for my BA in English. For one of my recent papers I had to reference a documentary available on Youtube. In my initial viewing I wrote down a couple of paraphrased quotes and foolishly neglected to record the timestamps.

For some reason, not all movies and TV shows on YouTube have transcripts available. This documentary did have closed captions though, so I knew that there must be a transcript some where. I really didn’t want to have to scrub through the whole 90 minute documentary looking for the quotes.

Finding the right words#

I pulled up the video, clicked the closed caption button, opened up the network tab of Chrome DevTools. Within a few seconds after pressing play, I saw a URL for “timedtext.”

I clicked open in new tab and downloaded the text file. Sure enough, all the words were there…
